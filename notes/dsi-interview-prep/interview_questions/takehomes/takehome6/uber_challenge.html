Uber Demand prediction challenge¶

First, let's load our dataset into a usable form. We'll be taking advantage of Pandas DataFrames in this Notebook due their great time-series and visualization abilities.

The first thing we do is import numpy and pandas, as well as set some pretty graphing settings. Once we've read in the JSON file as raw discrete time-series events, we bucket the data into demand by hour as a ts time-series by counting the discrete events within an hour span. Finally, I bring the TimeSeries into a DataFrame ( which will allow us to operform multi-dimensional analysis later ) and plot the resulting, binned data.

We'll use a support vector machine, trained on the hour and day of the week as inputs. While we have day and hour in continuous values, SVMs will train more accurately on highly-dimensional binary data. For that, we will encode the discrete hour and weekday values using a OneHotEncoder.

The extract_date_feature function creates appropriately encoded input data, along with an encoder that we can use later in vectorize_date_feature to automatically pre-process input data in future predictions.

In [15]: 
# Imports
from sklearn import svm
from sklearn import grid_search
from sklearn import metrics
from sklearn import preprocessing
from sklearn import cross_validation

# Preprocessing
def extract_date_feature(df):
    df['hour'] = df.index.map(lambda idx: idx.hour)
    df['weekday'] = df.index.map(lambda idx: idx.weekday())
    
    encoder = preprocessing.OneHotEncoder()
    encoder.fit_transform(df[['hour','weekday']])
    
    encoded = pd.concat([pd.get_dummies(df[col], prefix=col) for col in df.columns if not col in ['demand']], axis=1)
    return (pd.concat([df, encoded], axis=1), encoder)

def vectorize_date_feature(date, encoder):
    
    datestamp = pd.to_datetime(date)
    return encoder.transform([[datestamp.hour,datestamp.weekday()]]).toarray()[0]

# Encode the continuous data
dataset, encoder = extract_date_feature(df)

# We only want to train on encoded data
X_cols = [col for col in dataset.columns if not col in [u'demand', u'hour', u'weekday']]
y_cols = 'demand'

Once we have input data in a format that is ready to feed into our regressor, we split up the data into train and test datasets. Since we're doing time-series prediction, we'll attempt to simulate the problem of predicting future data by limiting our view to the first half of the dataset.

In order to optimize our regressor, we'll perform a grid search of the model's hyperparameters. This trains the model on a set of hyperparameter combinations and find the mose accurate model. We can access this optimized model via the best_estimator_ property.

Let's try a different method that ignores explicitely learning on the time of day and instead trains directly from historical data. To help visualize the dataset, we'll see what the two-week autocorrelation plot of the data looks like.

As we could have guessed, the data has a high autocorrelation with small lags, with smaller peaks one week and two weeks back. We've captured the lags associated with the highest correlation factor and can use those to construct an input dataset later on.

We'll now create some lagged data, where we derive the lags from the autocorrelation values before. The idea is that, given lagged data, we will be able to predict future demand. Note: high autocorrelation will not automatically give us the best inputs to train on, but is a good starting point!

# Preprocessing
window_size = lags.shape[0]
target_size = 1
input_size = window_size-target_size

shift = pd.concat([df['demand'].shift(n) for n in lags], axis=1).dropna()
target_columns = ['demand_0']
input_columns  = ['demand_{0}'.format(n) for n in xrange(target_size, window_size)]
shift.columns  = target_columns + input_columns

# Train/Test Split
split = int(shift.shape[0] * .8)
X,y = shift[input_columns], shift[target_columns]
X_train,X_test,y_train,y_test = X[0:split], X[split:], y[0:split], y[split:]

Now that we've created some lagged data, let's use another common machine learning model, Artificial Neural Networks, to create a predictor. We've already performed some preprocessing of the data by generating lagged samples, but ANNs also benefit from scaling the input data before training.

Because the ANN backpropegation takes so long, we pickle the the trained network so that it can be reused. To re-train, remove the net.pickle file.

    import pickle

    net = pickle.load(open('net.pickle','rb'))

    print(e)
    # Build and Train
    ds = SupervisedDataSet(inp=input_size, target=target_size)
    ds.setField('input', scaler.transform(X_train.values))
    ds.setField('target', scaler.transform(y_train.values))

    hidden_size = input_size * 5
    net = buildNetwork( input_size, hidden_size, target_size, bias = True, hiddenclass=TanhLayer)
    trainer = BackpropTrainer( net, ds )
    errors = trainer.trainUntilConvergence( verbose = True, validationProportion = 0.1, maxEpochs = 1000, continueEpochs = 5 )
    ret = pickle.dump(net, open('net.pickle','wb'))
